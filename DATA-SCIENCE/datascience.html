Data science


Intersección entre matemáticas, estadística, ciencias computacionales y conocimiento del dominio de la industria en el cual se usan datos para obtener información valiosa con 2 fines:


Crear productos basados en IA

Tomar decisiones y crear estrategias de negocio




No tiene por qué trabajar siempre con Big data, se puede trabajar con datos mas pequeños pero lo importante es que generen valor y sean confiables




Proceso


Pregunta o problema


Obtención de datos

Se puede hacer a través de mediciones directas o usando fuentes de internet

Limpieza y transformación de datos


Análisis de datos


Usar modelos de Machine learning


Integrar esos modelos a productos de software




Paso mas importante: Pregunta o problema

Crea los puentes para poder conectar con las BD

Automatización de los ETL


Validación de datos

Recolección, transformación, limpieza y almacenamiento de base de datos

Data Scientist


Incorpora datos a productos de ingeniería usando machine learning


Primeras personas en crear modelos y/o servicios de ML e interactuar con los modelos. Entrenan, evalúan y validan los modelos.




Diagnosticar el estado inicial de los datos cuando se entra a una empresa (hay fuentes de datos fiables? tienen alta volumetría? etc.)


Monitorear la precisión de los datos


Automatizar procesos de recolección y transformación de datos


Crear reportes de información de tableros




Data analyst


Toman los datos de las base de datos que los data engineer prepararon y los analizan para encontrar información de valor.

Insights que las personas del negocio puedan usar para tomar estrategias y acciones.


Identifica necesidades de información


Limpia y organiza los datos




Tener conocimiento del dominio es lo que usualmente hace la diferencia entre un data analyst y otro


Flujo de trabajo:


Parte de una pregunta o problema

Explorar queries en base a hipótesis

Crear visualización de la información

Comunicar hallazgos






Roles similares

Business Analyst & data visualization Analyst






ML Engineer


Trabaja de la mano con data scientist. Robustece el modelo y pone en producción. Despliega el modelo con ayuda de developers para que funcione en una aplicación y puedan ser utilizados en una interfaz por usuarios.


Escala modelos de IA


Genera evaluaciones de modelos de machine learning generados por el data scientist


Monitorear el desempeño y funcionalidad del modelo


La diferencia con el dta scientist es que este no pasa del generado del modelo de ML


Hacer escalable el modelo


Evaluación, validación y despliegue del modelo del data scientist

Necesidades



1. Etapa de recolección de los datos

: Los datos pueden venir de diversas fuentes dependiendo de la industria (páginas web, sensores, instrumentación…)

2. Etapa de Almacenamiento/Movimiento

: Los datos se deben movilizar y almacenar a través de una infraestructura o un pipeline.

3. Etapa de Exploración y transformación

: Se deben limpiar los datos, detectar errores y anomalías y prepararlos para un posterior análisis.

4. Etapa de Análisis

: Se buscan métricas, segmentaciones, agregaciones y distintas características para convertir los datos en información y posteriormente en conocimiento.

5. Etapa de IA

: Se generan predicciones y se visualiza el futuro de la organización.
