DHCP

Protocolo que asigna dirección IP a un dispositivo de forma aleatoria

  • Puerto server: 67
  • Puerto cliente: 68

Da la IP con el protocolo DORA

  • D: Discover: User solicita IP a la red, para eso hace un msg broadcast por el port: 68 a los servidores DHCP con IP = 0.0.0.0. Funciona porque todos estan conectados al mismo puerto
  • O: Offer: Cada Servidor envía una IP
  • R: Request: El servidor escoge una y manda un msg broadcast diciendo que IP acepta, mas DNS, netMask y gateway
  • A: Acknowledge: Server dice OK

Hay 2 formas:

Estática

Siempre asigna el mismo dispositivo. Identifica los dispositivos en base a su dirección MAC

Dinámica

Se le asigna un rango, y asigna aleatoriamente

Para que DHCP nunca se caiga existen unas practicas:

  • Tener varios DHCP servers con rangos distintos
  • DHCP Failover cluster:
    • Se tiene un Nodo (DHCP Server) pasivo que escucha lo que hace un nodo activo.
    • Un nodo activo 1-- * nodos pasivos
    • La ventaja del nodo pasivo es: No hay. La consecuencia es q los serivdores pueden asignar menos

DNS

  • Protocolo que traduce nombres en direcciones IP
  • DNS reverso: Verifica que una dirección efectivamente corresponda a una IP.
    • Los correos las usan para validar que una IP corresponde a un dominio
  • Se pueden configurar 1..* DNS por maquina. Pero si el primero no supo, los otros configurados no van a intentar buscar la equivalencia del nombre
  • Los dominios se guardan en cache: No aplica para root server
