Conectors and components (C & C)


Puede representar un subsistema

Cuando un componenente tiene una subestructura, se debe documentar la relación entre lo interno y lo externo



Elementos


Tanto componentes como conectores tendrán sus propias propiedades.


Ejemplos de propiedades

  • Realibility (probabilidad de fallo)
  • Performance (tipos de respuestas)
  • Resource requeriments (Requerimientos para almacenar y procesar data)
  • Functionalities
  • Security
  • Concurrency

Componente


Objeto que permite representar una acción o un evento dentro de un sistema


Tiene un conjunto de puertos con los cuales interactua con otros objetos


graph LR
A(components) --> |1..*| B(ports)


Puerto


Es una interfaz del componente. Es un punto que define como el componenete se comunica con el entorno


Se especifica el numero si es usado un socket


Estas difieren de interfaces ordinarias


Si se tienen multiples puertos, se pueden poner todos, o poner uno solo y al lado un [N], donde N es el total de puertos


No pueden ser duplicados


Conectores


Es una ruta entre 2 componentes

Estos si pueden ser duplicados

Son complejos por naturaleza

Tienen roles, es decir sus interfaces, los cuales dicen como podría ser usado por otros componentes

Ejemplo: Un pipe tiene un rol *writer* y otro *reader*



Ejemplos

  • Service invocation,
  • asynchronous message
  • Event multicast

Attachments

Se dan al conectar 2 roles


Call return style


Modelo computacional en el cual componentes proveen servicios que pueden ser invocados por otros componenentes.
