Threads


Es una serie de sentencias ejecutadas


Permite múltiples actividades dentro de un solo proceso o realizar tareas en segundo plano


Aprovecha los sistemas multiprocesador




Riesgos:


Se requiere buena coordinación para acceso a variables comunes.

Muchos subprocesos perjudicarán el rendimiento de la App y su mantenibilidad


threads en java


Tienen un grado de prioridad del 1 al 10, que puede ser modificado usando `setPriority()`.


Por default su prioridad es 5





Creación de threads



extends Thread


public class MyThread extends Thread {
   public void run(){  
    System.out.println("thread is running...");  
  } 
   public static void main(String[] args) {
     MyThread obj = new MyThread();
     obj.start();
}

implements runnable


public class MyThread implements Runnable {
   public void run(){  
     System.out.println("thread is running..");  
   }  
   public static void main(String[] args) {
     Thread t = new Thread(new MyThread());
     t.start();
}

Cual método es mejor?


La herencia implica que no se podrá abstraer mas objetos

La interfaz no soporta métodos que si soporta laherencia tales como suspend(), resume() etc.



Thread Scheduling


Ejecución de multiples hilos en una simple CPU en algún orden


notify()

: despierta el hilo que estaba en WAIT y lo pone en READY

notifyAll()

: despierta todos los hilos que estaban en WAIT y lo pone en READY

yield()

: Permite que otros subprocesos de la misma prioridad se ejecuten

nSleep()

: Pausa el hilo por determinado tiempo

isAlive()

: Verifica que siga vivo

join()

: Pone el hilo en espera hasta que el hilo con el que se enlazó muera.

Procesos






Es una instancia en ejecución de un programa al que el sistema asigna recursos como tiempo de CPU y memoria


cada proceso tiene al menos un hilo


Se forma un espacio de memoria para cada subproceso


Los subprocesos que se ejecutan dentro de un proceso comparten el conjunto común de recursos entre ellos que se asignan al proceso
